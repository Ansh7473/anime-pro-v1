## Goal
Replace the bloated ~1990-line watch page with a lean single-file `+page.svelte` (~500 lines) matching the reference layout, keeping the current Persimmon/`--net-*` color scheme. Delete all extra chrome. Keep video playback working via 2-3 providers (easily editable list).

## Reference layout (what the new page looks like)
Two-column flex (`max-w` container, stacks on mobile):
- **LEFT (flex-1):** video player → title + stats row (EP badge, ★rating, year, genres, views) → action row (Favorite / Share / Server picker dropdown / Report) → **Synopsis bento card** (expandable "Read More") → **"Select Episode" grid** with a season/range dropdown.
- **RIGHT (sticky, ~320-384px):** tabbed panel toggling **Live Chat** / **Comments**, message input pinned at bottom.

## What gets DELETED (chrome/bloat)
Bookmark banner, theater mode + lights-dim, keyboard-shortcuts modal, resume-progress prompt, save/update-history progress tracking, auto-next countdown, auto-skip/autoplay toggles, poster-color canvas extraction, PiP, rotation/fullscreen orchestration, related + recommendations rails, the 14-provider fan-out with priority ranking + grace-window logic, last-server localStorage pref, inline hls.js `initPlayer`/`tryNextSource`, audio-category grouping.

Sub-components no longer used will be removed: `WatchControlDeck`, `WatchTitleBar`, `WatchInfoCard`, `WatchEngagement`, `WatchServers`, `WatchSidebar`, `WatchPlayer` (their markup gets inlined into the lean page). `+page.server.ts` stays as-is.

## What gets KEPT / reused
- **Playback:** `$lib/components/VideoPlayer.svelte` for HLS/mp4 (it owns hls.js + proxy internally, just needs `src`, `poster`, `autoplay`, `onended`); a plain `<iframe>` branch for embed sources.
- **Community (library comps):** `LiveChat.svelte` (props `animeId`, `episode`, `isInline`, `onClose`), `CommentsSection.svelte` (`animeId`, `episode`), `ReactionsBar.svelte` (`animeId`, `episode`).
- `api.getAnime`, `getProxiedImage`, `BACKEND_URL`, auth store.

## New lean source loading (2-3 providers, editable)
Replace the whole engine with a small, obvious block:
```
const PROVIDERS = [
  api.getTatakaiSources,
  api.getHiAnimeSources,
  api.getNineAnimeSources,
]; // <- edit this list to enable/disable providers
```
`loadSources()`: fire all in parallel, collect into `sources`, pick first result as `selectedSource`, per-episode cache in a `Map`. Simple failover: if the active source errors, drop to the next in `sources`. Server-picker dropdown lets the user switch among whatever `sources` came back. Embed vs HLS decided by URL (`.m3u8`/`embed`).

## New page structure (single `+page.svelte`)
- `<script>`: props/`data`, `ep` synced from URL via `$effect`, `anime`/`episodes`/`sources`/`selectedSource` state, `sidebarTab` ('chat'|'comments'), `synopsisExpanded`, lean `loadSources`, `changeEp` (goto), `shareEpisode` (clipboard), `reportEpisode` (toast/alert), favorite toggle (local state; wire to existing watchlist api if present, else local).
- Template: the two-column reference layout described above, using `--net-*` tokens (Persimmon `#EC5800` accent for active tab / primary buttons, `--net-card-bg`, `--net-border`, `backdrop-filter`).
- `<style>`: fresh scoped CSS for `.watch-layout / .watch-main / .watch-aside` (sticky), `.stats-row`, `.action-row`, `.synopsis-card`, `.episode-grid`, `.sidebar-tabs / .sidebar-tab / .sidebar-panel / .sidebar-input`, responsive breakpoints (stack < 1024px). Icons via existing `lucide-svelte`.
